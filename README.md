## 概要

SupportFlowは、在宅スタッフによる問い合わせ対応・返信テンプレート管理・業務申し送りを一元管理できる業務支援システムです。  
小規模事業者や店舗運営における顧客対応業務を想定し、採用担当者が短時間で操作内容を確認できるよう設計しました。

外部DBや認証サービスは使用せず、**ブラウザのlocalStorageにデータを保存**するため、インストール不要でデモが可能です。

## 主な機能

- **ダッシュボード** — 今日の対応状況・未対応件数・スタッフ稼働状況を一覧表示
- **問い合わせ管理** — 受付・担当割り当て・ステータス管理・詳細メモ
- **返信テンプレート** — よく使う返信文をカテゴリ別に管理・ワンクリックコピー
- **業務申し送り** — スタッフ間の引き継ぎ事項を記録・確認済み管理
- **顧客管理** — 顧客情報・タグ管理・問い合わせ履歴
- **スタッフ管理** — 稼働ステータス・対応件数管理（管理者のみ）
- **レポート** — 種別別・スタッフ別集計・CSVエクスポート（管理者のみ）
- **設定** — 事業者情報・通知先・対応目標時間（管理者のみ）

## セットアップ方法

```bash
git clone https://github.com/ssk-t0/support-flow.git
cd support-flow
npm install
npm run dev
```

ブラウザで http://localhost:3000 を開いてください。  
初回アクセス時にサンプルデータが自動投入されます。

> `localhost:3000` はご自身のPC上でアプリを起動した場合にのみアクセス可能です。

### データのリセット

設定画面の「データをリセット」ボタン、またはブラウザのlocalStorageをクリアした後にページをリロードすると、サンプルデータが再投入されます。

## 工夫した点

### UX・デザイン
- **SaaS風の管理画面レイアウト** — 左サイドバー＋上部ヘッダーの定番構成で、初めて見る方でも迷わず操作できます
- **カラーコーディングされたステータスバッジ** — 未対応（赤）・対応中（黄）・完了（緑）で状況が一目でわかります
- **未対応・重要度「高」の強調表示** — テーブル行と申し送りカードで視覚的に目立たせています
- **ワンクリックコピー** — 返信テンプレートをコピーするボタンと「コピーしました」フィードバック

### 技術面
- **localStorage完結のデモ構成** — DBや認証サービス不要で、URLを共有するだけでデモ可能
- **サンプルデータの自動投入** — 初回アクセスで現実的なデモデータが投入され、すぐに操作感を確認できます
- **Next.js App Router対応** — localStorage/windowアクセスをuseEffect内に限定し、SSRエラーを回避
- **TypeScript厳格型定義** — `any`を極力使わず、型安全なコードベース
- **役割別メニュー制御** — 管理者とスタッフでサイドバーの表示項目を切り替え
- **CSVエクスポート** — BOM付きUTF-8でExcelでも文字化けしないCSV出力

## ポートフォリオとしてのアピールポイント

このプロジェクトは以下のスキルを実証するために作成しました：

1. **実務に近い要件定義からの実装** — 在宅スタッフ・小規模事業者の業務フローを想定した機能設計
2. **モダンなフロントエンド技術スタック** — Next.js 14 / TypeScript / Tailwind CSS の実践的な活用
3. **再利用可能なコンポーネント設計** — Modal・StatusBadge・FilterSelectなど汎用UIコンポーネントの分離
4. **ユーザビリティへの配慮** — 検索・フィルター・絞り込みの組み合わせ、空状態の表示など
5. **採用担当者がデモしやすい構成** — ワンクリックログイン・豊富なサンプルデータ・デモリセット機能
